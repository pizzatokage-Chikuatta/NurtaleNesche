アニメーションクリップ作成・パッチ
================================

重要：StreamingAssetsの注意
--------

`Assets\StreamingAssets`には直接読めるゲームデータが多く入っていますが、`Assets\StreamingAssets\AnimationClips`は例外です。`Assets\StreamingAssets\AnimationClips`内のファイルを直接編集しても、安全に配布できるアニメーションModにはなりません。

`Assets\StreamingAssets\AnimationClips`は、内蔵データの`clipId`、`trackId`、タイミング、位置情報などを確認するための参照用として使ってください。実際のアニメーションModファイルは、必ず`Mods\...`の中に置き、`mod.json`で`type: "animation"`を指定します。

このチュートリアルでは`type: "animation"`のModを説明します。

新しいclipIdを作ることもできますが、ネーシェのbody、eyemask、armbinderなどを差し替える場合は、基本的に`mergeMode: "patch"`を使います。これにより、変更していないtrackは内蔵データから継承されます。

敵キャラのcontroller slotを差し替えたい場合は、このチュートリアルでclip JSONの形を理解し、次の`35_アニメーションクリップオーバーライド.md`でmap.jsonを作ります。

最初に読むこと
--------

アニメーションModには大きく二つの流れがあります。

1. 敵キャラ差し替え：新しいclipIdを作り、`animationClipOverride`でcontroller slotに割り当てる。
2. ネーシェ差し替え：内蔵の`clipId`を維持し、`mergeMode: "patch"`で変更したいtrackだけを書く。

ネーシェの場合、全trackを巨大なJSONにコピーするのは推奨しません。変更したいtrackだけを持つ小さいpatchの方が、他のModと共存しやすいです。

参照ファイル：

~~~text
Mods\_Mod_Tutorial\JP\Reference_Data\Animation_Targets_And_Slots.md
Assets\StreamingAssets\AnimationClips
Assets\StreamingAssets\AnimationController_Detail
~~~

動作確認済みの例：

~~~text
Mods\Mod_Chikuatta\Animations\Example_02_Nesche_Normal_Body_Patch
Mods\Mod_Chikuatta\Animations\Example_03_Nesche_Normal_Eyemask_Patch
Mods\Mod_Chikuatta\Animations\Example_04_Nesche_Armbinder_Body_Patch
Mods\Mod_Chikuatta\Animations\Example_05_Nesche_Armbinder_Eyemask_Patch
~~~

これらは初期状態では無効です。テストする時だけ`mod.template.json`を`mod.json`へ変更してください。

ステップ1
--------

`Mods`の中に以下のフォルダー階層を作ります。

~~~text
Mods\Mod_YourName\Animations\Nesche Normal Eyemask Patch\Animations
~~~

`Mod_YourName`は自分のModフォルダー名に変更してください。

ステップ2
--------

以下のテンプレートをコピーします。

~~~text
Mods\mod.json templates\mod.template_actor_targeted.json
~~~

以下へ貼り付けます。

~~~text
Mods\Mod_YourName\Animations\Nesche Normal Eyemask Patch
~~~

コピーしたファイル名を以下へ変更します。

~~~text
mod.json
~~~

ステップ3
--------

`mod.json`を開き、安全なtrack patch用に以下の形にします。

~~~json
{
  "experimentalApiVersion": "v1",
  "type": "animation",
  "target": "Nesche",
  "variant": "",
  "entry": "Animations",
  "mergeMode": "patch"
}
~~~

項目の意味：

1. `type`: アニメーションクリップJSONなので`animation`。
2. `target`: アニメーションtarget ID。ネーシェは`Nesche`。
3. `variant`: 空文字は通常/baseのネーシェ。
4. `entry`: `mod.json`の横にあるフォルダー名と一致させる。
5. `mergeMode`: trackだけを差し替える場合は`patch`。

patroller idをtargetにしてはいけません。これは間違いです。

~~~text
patroller.goblin.stage_06
~~~

ステップ4
--------

patchしたい内蔵clipを正確に選びます。

通常idleのネーシェなら、この参照ファイルを見ます。

~~~text
Assets\StreamingAssets\AnimationClips\Nesche\Actions\Normal\Idle E_animations.json
~~~

この中にある内蔵`clipId`を確認します。

~~~text
nesche.actions.normal.nesche_normal_idle_e
~~~

最初のテストでは、ネーシェpatch用に新しい`clipId`を作らないでください。既存clipをpatchするため、内蔵`clipId`をそのまま使います。

ステップ5
--------

`Animations`フォルダー内に以下のファイルを作ります。

~~~text
my_nesche_normal_eyemask_animations.json
~~~

ファイル名は必ず以下で終わる必要があります。

~~~text
_animations.json
~~~

`_animations.json`で終わらない場合、ローダーは読み込みません。

ステップ6
--------

`my_nesche_normal_eyemask_animations.json`に以下を書きます。

~~~json
{
  "clips": [
    {
      "name": "Nesche Normal Eyemask Patch",
      "clipId": "nesche.actions.normal.nesche_normal_idle_e",
      "tracks": [
        {
          "trackId": "eyemask",
          "spriteFrames": [
            {
              "spriteName": "My_Eyemask_0",
              "time": 0.0,
              "duration": 0.3
            },
            {
              "spriteName": "My_Eyemask_1",
              "time": 0.3,
              "duration": 0.3
            }
          ]
        }
      ]
    }
  ]
}
~~~

このpatchは`trackId: "eyemask"`だけを持ちます。

`mergeMode`が`patch`なので、body、mouthpiece、armbinder、leg shacklesなど他のtrackは内蔵データから継承されます。

ステップ7
--------

JSONファイルの横にspriteフォルダーを作ります。patch Modでは`clipId`と同じフォルダー名にしてください。

~~~text
Mods\Mod_YourName\Animations\Nesche Normal Eyemask Patch
  mod.json
  Animations
    my_nesche_normal_eyemask_animations.json
    nesche.actions.normal.nesche_normal_idle_e
      My_Eyemask_0.png
      My_Eyemask_1.png
~~~

重要ルール：

1. spriteフォルダー名は`clipId`と一致させる。
2. PNGファイル名は`spriteName`と一致させる。
3. `spriteName`には`.png`を書かない。
4. `My_Eyemask_0`なら実ファイルは`My_Eyemask_0.png`。

ステップ8
--------

track名を理解します。

よく使うtrack id：

1. `body`: メインbodyのsprite。
2. `cloth`: 服のoverlay。
3. `eyemask`: アイマスクoverlay。
4. `mouthpiece`: 口枷overlay。
5. `armbinder`: armbinder/restraint overlay。
6. `yoke`: yoke系overlay。clipによって存在するか変わります。

track idは勘で書かないでください。`Assets\StreamingAssets\AnimationClips`の似た内蔵ファイルから正確な`trackId`をコピーしてください。

ステップ9
--------

`spriteFrames`を理解します。

各frameに必要なもの：

1. `spriteName`: `.png`なしのPNGファイル名。
2. `time`: そのspriteが始まる秒数。
3. `duration`: そのspriteを表示する秒数。

最初のModでは内蔵ファイルの`time`と`duration`をコピーし、`spriteName`とPNGだけ変更するのが安全です。

ステップ10
--------

`transforms`を理解します。

transform keyは以下の形です。

~~~json
{
  "time": 0.0,
  "value": 0.637
}
~~~

これは有効です。

~~~json
{
  "transforms": {
    "position": {
      "x": [ { "time": 0.0, "value": 0.0 } ],
      "y": [ { "time": 0.0, "value": 0.0 } ],
      "z": []
    }
  }
}
~~~

これは間違いです。

~~~text
"position": { "x": [0.5], "y": [0.3], "z": [0] }
~~~

単純なpatch Modでは`transforms`を省略できます。その場合、ゲームは内蔵/full replacement側のtransformを継承します。

ステップ11
--------

variantを理解します。

ネーシェには複数のanimation variantがあります。同じtrack名でもvariantが違えば別物です。

例：

1. 通常/baseネーシェは`variant: ""`。
2. Armbinderネーシェは`variant: "armbinder"`。
3. Cuff/collar/chain系は`variant: "ccc"`の場合があります。
4. Yoke系は`variant: "yoke_bondage"`の場合があります。

通常eyemask patchはarmbinder clipには効きません。armbinder eyemask patchは通常clipには効きません。

normalとarmbinderの両方にeyemask skinを対応させたい場合は、二つのpatchを作ってください。

1. `variant: ""`と通常用`clipId`のpatch。
2. `variant: "armbinder"`とarmbinder用`clipId`のpatch。

ステップ13
--------

ゲームを起動し、以下を開きます。

~~~text
Main Menu -> Mods -> Mod Report
~~~

その後、ゲーム内で確認します。

期待結果：

1. Mod Reportでanimation Modとして読み込まれている。
2. 意図したclip/trackだけが変わる。
3. 他のtrackは正常に表示される。
4. 他variantは、そのvariant用patchを作っていない限り変わらない。

よくあるミス
--------

1. `Assets\StreamingAssets\AnimationClips`を直接編集する。ここは参照用です。
2. patroller idを`target`にする。`Nesche`や`Orc_AC Edit`などのanimation target idを使う。
3. trackだけのネーシェModなのに`mergeMode: "patch"`を書き忘れる。
4. `variant: ""`でarmbinder clipも変わると思い込む。
5. `body`や`eyemask`だけ変えたいのに全trackを入れてしまう。
6. transform配列に数値だけを書く。`{ "time": ..., "value": ... }`の形が必要です。
7. `spriteName`とPNGファイル名が一致していない。
