新しいアニメーションクリップ作成
================================

このチュートリアルでは、Goblin用の新しいアニメーションクリップJSONとPNGを作ります。

重要：このチュートリアルだけではゲーム内のアニメーションは差し替わりません。ここでは「読み込める新しいclipId」を作るだけです。実際にGoblinへ使わせるには、次の「35_アニメーションクリップオーバーライド.md」を使います。

ステップ1
--------

Modsの中に以下のフォルダー階層を作る。

~~~text
Mods\Mod_YourName\Animations\Goblin Idle Clip\Animations
~~~

Mod_YourNameは自分のModフォルダー名に変更してください。

ステップ2
--------

以下のテンプレートをコピーする。

~~~text
Mods\mod.json templates\mod.template_actor_targeted.json
~~~

以下へ貼り付ける。

~~~text
Mods\Mod_YourName\Animations\Goblin Idle Clip
~~~

ファイル名をmod.jsonへ変更する。

ステップ3
--------

mod.jsonを開いて中身を以下にする。

~~~json
{
  "experimentalApiVersion": "v1",
  "type": "animation",
  "target": "Goblin_AC Edit",
  "variant": "",
  "entry": "Animations"
}
~~~

項目の意味：

1. "type": "animation" はアニメーションクリップJSONを読み込む指定。
2. "target": "Goblin_AC Edit" はアニメーションセットID。patroller.goblinではない。
3. "variant": "" は基本バリアント。
4. "entry": "Animations" はmod.jsonの横にあるフォルダー名と一致させる。

ステップ4
--------

Animationsフォルダーの中に以下のファイルを作る。

~~~text
my_goblin_idle_animations.json
~~~

ファイル名は必ず以下で終わる必要があります。

~~~text
_animations.json
~~~

実際のModでは、StreamingAssets\AnimationClips から近い built-in の _animations.json をコピーして改造する方が安全です。ゼロから書くより、まず動いているJSONの形を保つ方がミスが少ないです。

ステップ5
--------

my_goblin_idle_animations.jsonの中身を以下にする。

~~~json
{
  "clips": [
    {
      "name": "mod.yourname.goblin_idle",
      "clipId": "mod.yourname.goblin_idle",
      "frameRate": 100.0,
      "pixelsPerUnit": 54.0,
      "loopTime": true,
      "tracks": [
        {
          "trackId": "body",
          "spriteFrames": [
            {
              "spriteName": "My_Goblin_Idle_0",
              "time": 0.0,
              "duration": 0.3
            },
            {
              "spriteName": "My_Goblin_Idle_1",
              "time": 0.3,
              "duration": 0.3
            }
          ],
          "transforms": {
            "position": { "x": [], "y": [], "z": [] },
            "rotation": { "x": [], "y": [], "z": [] },
            "scale": { "x": [], "y": [], "z": [] },
            "color": { "r": [], "g": [], "b": [], "a": [] }
          }
        }
      ],
      "transforms": {
        "position": { "x": [], "y": [], "z": [] },
        "rotation": { "x": [], "y": [], "z": [] },
        "scale": { "x": [], "y": [], "z": [] },
        "color": { "r": [], "g": [], "b": [], "a": [] }
      },
      "events": []
    }
  ]
}
~~~

ステップ6
--------

Animationsフォルダーの中に、clipIdと同じ名前のフォルダーを作る。

~~~text
mod.yourname.goblin_idle
~~~

その中にPNGを入れる。

~~~text
My_Goblin_Idle_0.png
My_Goblin_Idle_1.png
~~~

spriteNameには.pngを書かない。spriteNameがMy_Goblin_Idle_0なら、実ファイルはMy_Goblin_Idle_0.pngです。

ステップ7
--------

trackIdを理解する。

trackIdは「このトラックがどの描画レイヤーを動かすか」です。自由な名前ではありません。対象アニメーションセットに実在するrenderer trackと一致している必要があります。

候補例：

~~~text
body
cloth
eyemask
mouthpiece
armbinder
leg_shackles
rotor
shadow
~~~

重要：

1. このGoblin例では基本的に"body"を使う。
2. Playerやfriendly captive girlでは、"cloth", "eyemask", "mouthpiece"などが使える場合がある。
3. ただし対象がそのtrackを持っていない場合、そのtrackは正しく表示されない。
4. 有効なtrackIdは、同じactorのbuilt-in _animations.jsonを開き、"trackId"で検索して確認する。

ステップ8
--------

transformsの配列を理解する。

これは正しい。「positionを動かさない」という意味：

~~~text
"position": { "x": [], "y": [], "z": [] }
~~~

これは間違い：

~~~text
"position": { "x": [0.5], "y": [0.3], "z": [0] }
~~~

配列の中は数値そのものではなく、以下のようなkey objectです。

~~~text
"position": {
  "x": [
    { "time": 0.0, "value": 0.0 },
    { "time": 0.3, "value": 0.05 }
  ],
  "y": [],
  "z": []
}
~~~

"time"は秒、"value"はその時点の値です。

ステップ9
--------

ゲームを起動し、Main Menu -> Mods -> Mod Reportで読み込まれているか確認する。

注意：この時点ではGoblinの見た目はまだ変わりません。次の35番でGoblin_Idling.mainをこのclipIdへ差し替えます。

よくあるミス
--------

1. targetにpatroller.goblinを書く。正しくはGoblin_AC Edit。
2. JSONファイル名が_animations.jsonで終わっていない。
3. PNGをclipIdフォルダーではなくJSONの横へ置いている。
4. spriteNameに.pngを書いている。
5. transformsの配列に数値を直接入れている。
6. Goblinにeyemaskなど存在しないtrackIdを入れている。
