アニメーターコントローラー
================================

このチュートリアルではanimatorController modの安全な編集方法を説明します。普通のアニメーション差し替えなら、まず35番のanimationClipOverrideを使ってください。

animatorControllerは強力ですが壊しやすいです。controller全体を差し替えるため、部分パッチではありません。

ステップ1
--------

Modsの中に以下のフォルダー階層を作る。

~~~text
Mods\Mod_YourName\Animations\Goblin Controller\Animator Controllers
~~~

ステップ2
--------

mod.template_actor_targeted.jsonをコピーして、mod.jsonへ変更する。

mod.jsonの中身：

~~~json
{
  "experimentalApiVersion": "v1",
  "type": "animatorController",
  "target": "Goblin_AC Edit",
  "variant": "",
  "entry": "Animator Controllers"
}
~~~

ステップ3
--------

以下のbuilt-in controllerをコピーする。

~~~text
StreamingAssets\AnimationController_Detail\Goblin\Goblin_AC Edit_controller.json
~~~

以下へ貼り付ける。

~~~text
Mods\Mod_YourName\Animations\Goblin Controller\Animator Controllers
~~~

ファイル名は必ず_controller.jsonで終わる必要があります。

ステップ4
--------

最初のテストではclipIdを一つだけ変える。

以下を探す。

~~~json
{
  "stateId": "Goblin_Idling",
  "clips": {
    "main": "goblin.goblin_idling"
  },
  "tuning": {},
  "motionParameters": {}
}
~~~

"goblin.goblin_idling"だけを変更する。

~~~json
{
  "stateId": "Goblin_Idling",
  "clips": {
    "main": "mod.yourname.goblin_idle"
  },
  "tuning": {},
  "motionParameters": {}
}
~~~

ステップ5
--------

重要なcontroller項目：

1. "id": controllerのID。
2. "entryStateId": controller開始時のstate。
3. "stateMotionBindings": stateとclip slotの対応。
4. "stateId": runtime codeが呼ぶstate key。
5. "clips": slot名からclipIdへの対応。
6. "parameters": speedなどruntime値。
7. "layers": stateのグループ。
8. "states": 詳細なstate定義。
9. "blendTree": walk/runなどのブレンド。
10. "transitions": 自動遷移。

安全ルール
--------

最初のcontroller modでは以下を守る。

1. state idを変えない。
2. parametersを消さない。
3. transitionsを消さない。
4. layersを消さない。
5. blendTreeを触らない。
6. clipIdを一つだけ変更する。

避けるべき編集
--------

1. "parameters": runtime codeが名前を参照している可能性がある。
2. "layers": 消すとstateグループごと消える。
3. "transitions": 消すとactorが停止する可能性がある。
4. "blendTree": walk/runが壊れやすい。
5. "entryStateId": 間違えると透明/停止状態で始まる可能性がある。

よくある症状
--------

1. Goblinが消える：clipIdのclipが描画できない。
2. Goblinが歩かない：walk/run stateまたはspeed parameterが壊れている。
3. ずっとIdle：transitionやruntime呼び出しstateIdが壊れている。
4. Modは読み込まれるが変化しない：target違い、または他Modに負けている。
