アニメーションクリップオーバーライド
================================

重要：StreamingAssetsの注意
--------

`Assets\StreamingAssets`には直接読めるゲームデータが多く入っていますが、`Assets\StreamingAssets\AnimationClips`は例外です。`Assets\StreamingAssets\AnimationClips`内のファイルを直接編集しても、安全に配布できるアニメーションModにはなりません。

`Assets\StreamingAssets\AnimationClips`は、内蔵データの`clipId`、`trackId`、タイミング、位置情報などを確認するための参照用として使ってください。実際のアニメーション差し替えModは、必ず`Mods\...`の中に置き、`mod.json`で`type: "animation"`または`type: "animationClipOverride"`を指定します。

このチュートリアルでは`type: "animationClipOverride"`のModを説明します。

既存のanimator controller slotに別のclipIdを再生させたい時に使います。敵キャラの通常のアニメーション差し替えはこの流れです。

ネーシェのbodypart/accessoryの一部trackだけをpatchする場合は、このチュートリアルではなく`34_新しいアニメーションクリップ作成.md`の`mergeMode: "patch"`を使ってください。

作るもの
--------

このチュートリアルではOrcの歩きslotを差し替えます。

~~~text
Orc_WalkRun.walk
~~~

差し替え先の新しいclipId：

~~~text
mod.chikuatta.example.orc_walk_body
~~~

動作確認済みの例：

~~~text
Mods\Mod_Chikuatta\Animations\Example_01_Orc_Body_Walk_Override
~~~

この例は初期状態では無効です。テストする時だけ`mod.template.json`を`mod.json`へ変更してください。

ステップ1
--------

`Mods`の中に以下のフォルダー階層を作ります。

~~~text
Mods\Mod_YourName\Animations\Orc Walk Override\Animation Clip Overrides
~~~

ステップ2
--------

以下のテンプレートをコピーします。

~~~text
Mods\mod.json templates\mod.template_actor_targeted.json
~~~

以下へ貼り付けます。

~~~text
Mods\Mod_YourName\Animations\Orc Walk Override
~~~

ファイル名を以下へ変更します。

~~~text
mod.json
~~~

ステップ3
--------

`mod.json`を開き、以下にします。

~~~json
{
  "experimentalApiVersion": "v1",
  "type": "animationClipOverride",
  "target": "Orc_AC Edit",
  "variant": "",
  "entry": "Animation Clip Overrides"
}
~~~

項目の意味：

1. `type`: controller slot差し替えmapなので`animationClipOverride`。
2. `target`: animation target/controller ID。Orcは`Orc_AC Edit`。
3. `variant`: 空文字はbase/default controller variant。
4. `entry`: `mod.json`の横にあるフォルダー名と一致させる。

patroller data idをtargetにしてはいけません。これは間違いです。

~~~text
patroller.orc.stage_06
~~~

ステップ4
--------

`Animation Clip Overrides`の中に以下のファイルを作ります。

~~~text
map.json
~~~

`map.json`に以下を書きます。

~~~json
{
  "priority": 100,
  "overrides": {
    "items": [
      {
        "clipSlot": "Orc_WalkRun.walk",
        "clipId": "mod.yourname.orc_walk_body"
      }
    ]
  }
}
~~~

項目の意味：

1. `priority`: 同じslotを複数Modが差し替える場合、大きい方が勝つ。
2. `overrides.items`: slot差し替えのリスト。
3. `clipSlot`: 差し替えるcontroller slot。
4. `clipId`: 代わりに再生するanimation clip id。

ステップ5
--------

`clipSlot`を理解します。

形式は以下です。

~~~text
stateId.slotName
~~~

つまりこれ：

~~~text
Orc_WalkRun.walk
~~~

意味：

1. `stateId`は`Orc_WalkRun`。
2. `slotName`は`walk`。

slotを探すには内蔵controller JSONを開きます。

~~~text
Assets\StreamingAssets\AnimationController_Detail\Orc\Orc_AC Edit_controller.json
~~~

`stateMotionBindings`を探します。以下の形なら`Orc_WalkRun.walk`と`Orc_WalkRun.run`が存在します。

~~~json
{
  "stateId": "Orc_WalkRun",
  "clips": {
    "walk": "orc.orc_walk",
    "run": "orc.orc_run"
  }
}
~~~

ステップ6
--------

同じentryフォルダー内に差し替え用animation clipを作ります。

`Animation Clip Overrides`内に以下のファイルを作ります。

~~~text
my_orc_walk_body_animations.json
~~~

ファイル名は必ず`_animations.json`で終わる必要があります。

`map.json`で指定した`clipId`と同じclipを入れます。

~~~json
{
  "clips": [
    {
      "name": "mod.yourname.orc_walk_body",
      "clipId": "mod.yourname.orc_walk_body",
      "frameRate": 100.0,
      "pixelsPerUnit": 54.0,
      "loopTime": true,
      "tracks": [
        {
          "trackId": "body",
          "spriteFrames": [
            {
              "spriteName": "My_Orc_Walk_0",
              "time": 0.0,
              "duration": 0.15
            },
            {
              "spriteName": "My_Orc_Walk_1",
              "time": 0.15,
              "duration": 0.15
            }
          ]
        }
      ],
      "events": []
    }
  ]
}
~~~

ステップ7
--------

新しい`clipId`と同じ名前のspriteフォルダーを作ります。

~~~text
Mods\Mod_YourName\Animations\Orc Walk Override
  mod.json
  Animation Clip Overrides
    map.json
    my_orc_walk_body_animations.json
    mod.yourname.orc_walk_body
      My_Orc_Walk_0.png
      My_Orc_Walk_1.png
~~~

重要ルール：

1. spriteフォルダー名は`clipId`と一致させる。
2. PNGファイル名は`spriteName`と一致させる。
3. `spriteName`には`.png`を書かない。
4. `map.json`のclipIdと`_animations.json`のclipIdを一致させる。

ステップ8
--------

実際にOrc bodyを差し替える場合は、内蔵Orc歩きアニメーションからタイミングをコピーします。

~~~text
Assets\StreamingAssets\AnimationClips\Orc\Orc_Walk_animations.json
~~~

最初は`body` trackのtimingを保ち、読み込みが成功してからPNGを差し替えるのが安全です。

ステップ9
--------

ゲームを起動し、以下を開きます。

~~~text
Main Menu -> Mods -> Mod Report
~~~

その後、Orcがいるステージで歩きアニメーションを確認します。

期待結果：

1. Mod Reportにclip overrideが読み込まれる。
2. Orcの歩きが差し替えspriteになる。
3. Orcのrun/idle/attackは、別途overrideしていなければ変わらない。

よくあるミス
--------

1. `target: "patroller.orc.stage_06"`にする。正しくは`Orc_AC Edit`。
2. built-in clip idを`clipSlot`にしてしまう。`clipSlot`は`stateId.slotName`です。
3. `clipSlot: "orc.orc_walk"`と書く。間違いです。`Orc_WalkRun.walk`を使います。
4. `map.json`にclipIdを書いたが、そのclipをどこにも用意していない。
5. `map.json`を忘れる。`animationClipOverride`には必須です。
6. 一つのslot overrideでOrcの全アニメーションが変わると思い込む。slotごとにmapが必要です。
7. ネーシェのeyemaskなど一つのtrackだけを変えたいのにこの流れを使う。チュートリアル34の`mergeMode: "patch"`を使ってください。
