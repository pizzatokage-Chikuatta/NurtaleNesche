インタラクションロードアウト
================================
チェインポイントやアイテムなどが持つインタラクション選択肢リストのModを作ります。表示候補の組み合わせを変えるためのデータです。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_Chikuatta\Advanced\Mod Interaction Loadouts\Interaction Loadouts\ChainPoints

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Interaction Loadouts'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。
```json
{
  "experimentalApiVersion": "v1",
  "type": "interactionLoadouts",
  "entry": "Interaction Loadouts",
  "overwriteExisting": true
}
```
①今回はインタラクションロードアウトのModなので"type"はこの値にする。
②"entry"は横のフォルダー名と一致しないといけない。

ステップ4
--------
以下のフォルダーから'ChainPoints\guillotine.json'をコピーし、
StreamingAssets\Interaction\Loadouts
以下へ張り付ける。
Mods\Mod_Chikuatta\Advanced\Mod Interaction Loadouts\Interaction Loadouts

ステップ5
--------
'ChainPoints\guillotine.json'の中身を修正する。
1. "id": このロードアウトのId。既存を書き換える場合は変えてはいけない。
2. "commands"または選択肢リスト: 表示/実行候補にするインタラクションIdの配列。
3. "condition"系の項目がある場合: 表示条件。既存データを参考にする。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

注意
--------
1. ロードアウトにIdを書いただけでは、そのIdの処理が新規作成されるわけではない。
2. 存在しないコマンドIdを入れると、選択肢が出ない/押しても動かない原因になる。
3. ChainPoint、Item、その他対象でフォルダーが分かれているため、コピー元の階層を維持する。

置き換えリストの注意
--------

このファイルは「少し追加するパッチ」ではなく、リスト全体の置き換えです。同じ"id"を上書きする場合、最終的に使われるリストはMod側に書いたリストになります。残したいbuilt-in項目は消さずに残してください。

コピーしてから少しずつ変えるルール
--------

この上級者向けデータは、まずbuilt-inファイルをコピーして一項目ずつ変更してください。空のJSONから作ると、runtime codeや他registryへの隠れた参照を見落としやすいです。
