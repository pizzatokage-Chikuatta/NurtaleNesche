アイテム選択リクエスト定義
================================
「どのアイテムを選ばせるか」を決めるリクエスト定義のModを作ります。インタラクションからアイテム選択UIを出す時に使われます。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_Chikuatta\Advanced\Mod Item Selection Requests\Item Selection Requests

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Item Selection Requests'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。
```json
{
  "experimentalApiVersion": "v1",
  "type": "itemSelectionRequestDefinitions",
  "entry": "Item Selection Requests",
  "overwriteExisting": true
}
```
①今回はアイテム選択リクエスト定義のModなので"type"はこの値にする。
②"entry"は横のフォルダー名と一致しないといけない。

ステップ4
--------
以下のフォルダーから'item_selection_request_definitions.json'をコピーし、
StreamingAssets\Interaction\Item Selection Requests
以下へ張り付ける。
Mods\Mod_Chikuatta\Advanced\Mod Item Selection Requests\Item Selection Requests

ステップ5
--------
'item_selection_request_definitions.json'の中身を修正する。
1. "id": リクエスト定義Id。既存を書き換える場合は変えてはいけない。
2. "allowedItemIds"系: 選択可能なアイテムIdリスト。
3. "allowedEquipmentIds"系: 選択可能な装備Idリスト。
4. "statusEffectId"や"category"系がある場合: 何を対象にするかの絞り込み。
5. "message"系がある場合: UI表示用テキスト/キー。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

注意
--------
1. アイテム/装備IdはReference_Dataフォルダーを参照する。
2. 選択可能リストを空にすると、UIは開いても選べるものが無い状態になりやすい。
3. Interaction Loadout側と繋がるデータなので、片方だけ編集すると動かない場合がある。

コピーしてから少しずつ変えるルール
--------

この上級者向けデータは、まずbuilt-inファイルをコピーして一項目ずつ変更してください。空のJSONから作ると、runtime codeや他registryへの隠れた参照を見落としやすいです。
