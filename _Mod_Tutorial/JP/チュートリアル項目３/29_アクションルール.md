アクションルール
================================
ActionScheduler2の行動をブロック/許可する条件ルールのModを作ります。これは行動AIに直接影響するため、初心者向けではありません。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_Chikuatta\Advanced\Mod Action Rules\Action Rules

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Action Rules'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。
```json
{
  "experimentalApiVersion": "v1",
  "type": "actionRules",
  "entry": "Action Rules",
  "overwriteExisting": true
}
```
①今回はアクションルールのModなので"type"はこの値にする。
②"entry"は横のフォルダー名と一致しないといけない。

ステップ4
--------
以下のフォルダーから'profile.player.action_rules.json'をコピーし、
StreamingAssets\Action\Action Rules
以下へ張り付ける。
Mods\Mod_Chikuatta\Advanced\Mod Action Rules\Action Rules

ステップ5
--------
'profile.player.action_rules.json'の中身を修正する。
1. "profileId": このルール集の対象Id。基本変えない。
2. "rules": 実際のルール配列。
3. "passCheckId": このルールが判定する条件Id。
4. "blockIfAny": どれか一つでも当てはまるとブロックする条件。
5. "blockIfMissing": 指定した条件が無い場合ブロックする条件。
6. "blockIfNeither": 指定したどちらの条件も無い場合ブロックする条件。
7. "forcePassIfAny": どれか一つでも当てはまると強制的に通す条件。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

注意
--------
1. 一つのルール変更でAI行動が大きく変わるため、同時に複数箇所を編集しない。
2. ルールを削除するより、まずは条件を一つだけ追加/変更してテストする。
3. PatrollerDataのTask/TaskSetが正しくても、Action Rulesでブロックされると行動しない。

置き換えリストの注意
--------

このファイルは「少し追加するパッチ」ではなく、リスト全体の置き換えです。同じ"id"を上書きする場合、最終的に使われるリストはMod側に書いたリストになります。残したいbuilt-in項目は消さずに残してください。

コピーしてから少しずつ変えるルール
--------

この上級者向けデータは、まずbuilt-inファイルをコピーして一項目ずつ変更してください。空のJSONから作ると、runtime codeや他registryへの隠れた参照を見落としやすいです。
