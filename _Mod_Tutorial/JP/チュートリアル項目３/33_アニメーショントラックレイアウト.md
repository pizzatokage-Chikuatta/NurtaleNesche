アニメーショントラックレイアウト
================================
JSONアニメーションがどのSpriteRenderer/トラックに描画されるかを定義するModを作ります。見た目の土台を触るため、かなり注意が必要です。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_Chikuatta\Advanced\Mod Animation Track Layouts\Animation Track Layouts

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Animation Track Layouts'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。
```json
{
  "experimentalApiVersion": "v1",
  "type": "animationTrackLayouts",
  "entry": "Animation Track Layouts",
  "overwriteExisting": true
}
```
①今回はアニメーショントラックレイアウトのModなので"type"はこの値にする。
②"entry"は横のフォルダー名と一致しないといけない。

ステップ4
--------
以下のフォルダーから'nesche_player_track_layout.json'をコピーし、
StreamingAssets\AnimationTrackLayouts
以下へ張り付ける。
Mods\Mod_Chikuatta\Advanced\Mod Animation Track Layouts\Animation Track Layouts

ステップ5
--------
'nesche_player_track_layout.json'の中身を修正する。
1. "id": トラックレイアウトId。既存を書き換える場合は変えてはいけない。
2. "tracks": 使用する描画トラック一覧。
3. "trackId": アニメーションJSON側から参照されるトラック名。
4. "rendererPath"系: Unity prefab内のSpriteRendererへのパス。
5. "defaultVisible"系: 初期表示状態。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

注意
--------
1. rendererPathを間違えると、キャラや一部パーツが見えなくなる。
2. Player/Patroller/Friendly Captive Girlはトラック構造が違うため、別キャラのレイアウトを雑に流用しない。
3. アニメーション差し替えより先に触るデータではない。基本は既存レイアウトをコピーして最小変更にする。

コピーしてから少しずつ変えるルール
--------

この上級者向けデータは、まずbuilt-inファイルをコピーして一項目ずつ変更してください。空のJSONから作ると、runtime codeや他registryへの隠れた参照を見落としやすいです。
