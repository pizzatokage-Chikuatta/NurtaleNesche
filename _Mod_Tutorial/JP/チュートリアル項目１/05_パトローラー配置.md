パトローラー配置
================================
ステージ開始時に配置されるパトローラーのModを作ります。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_｛個人のニックネーム｝\Patrollers\Mod Initial Spawns\Initial Spawns\Stage 1

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Initial Spawns'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。
```json
{
  "experimentalApiVersion": "v1",
  "type": "patrollerSpawns",
  "entry": "Initial Spawns",
  "overwriteExisting": true
}
```
①今回はパトローラー配置のModなので"type"は"patrollerSpawns"である。
②"entry"は横のフォルダー名と一致しないといけない。

ステップ4
--------
以下のフォルダーから'patroller_spawn.orc.stage_01.json'などをコピーし、
StreamingAssets\Patrollers\Initial Spawns\Stage 1
以下へ張り付ける。
Mods\Mod_Chikuatta\Patrollers\Mod Initial Spawns\Initial Spawns\Stage 1

ステップ5
--------
コピーしたパトローラー配置Jsonの中身を修正する。
①"id": 既存配置を書き換える場合は変えてはいけない。新規追加する場合は他と被らないIdにする。
②"patrollerId": スポーンさせるパトローラーDataのId。例: "patroller.orc.stage_01"など。
③"enabled": falseの場合この配置は使われない。
④"spawnRegionId": スポーンする地域Id。Reference_Dataフォルダー内の「NurtaleNesche_Modding_Region_UUIDs.md」を参照。
⑤"patrolRegionIds": このパトローラーが巡回候補にする地域Idリスト。
⑥"xOffsetFromRegionCenter": 地域中心から左右にどれだけズラして配置するか。プラスで右、マイナスで左。
⑦"yOffsetFromGround": 地面から上下にどれだけズラすか。浮く/埋まる時だけ調整。
⑧"useSpawnRegionCheckpointPosition": trueの場合、地域中心ではなくRegionのCheckpoint位置に配置する。この場合"xOffsetFromRegionCenter"は基本効かない。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

備考
--------
①パトローラー配置Jsonを作っただけではステージに出ない場合がある。その時はパトローラー配置ロードアウトにもIdを追加する。
②"patrollerId"はPatrollers\Data内のデータIdを参照する。
