アイテム箱のステージ配置
================================
アイテム箱のステージ配置のModを作ります。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_｛個人のニックネーム｝\Lootings\Item Boxes\Mod Stage Initial Spawns\Stage Initial Spawns\Stage 01

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Stage Initial Spawns'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。
```json
{
  "experimentalApiVersion": "v1",
  "type": "itemBoxSpawns",
  "entry": "Stage Initial Spawns",
  "overwriteExisting": true
}
```
①今回はアイテム箱配置のModなので"type"は"itemBoxSpawns"である。
②"entry"は横のフォルダー名と一致しないといけない。

ステップ4
--------
以下のフォルダーから'item_box_spawn.stage_01.001.itembox.json'をコピーし、
StreamingAssets\Lootings\Item Boxes\Stage Initial Spawns\Stage 01
以下へ張り付ける。
Mods\Mod_Chikuatta\Lootings\Item Boxes\Mod Stage Initial Spawns\Stage Initial Spawns\Stage 01

ステップ5
--------
'item_box_spawn.stage_01.001.itembox.json'の中身を修正する。
①"id": 既存の箱を書き換える場合は変えてはいけない。新しい箱として追加する場合は他と被らないIdにする。
②"enabled": falseの場合この箱はスポーンしない。
③"boxPrefabId": アイテム箱のUnityオブジェクトId。基本変えない。
④"spawnRegionId": スポーン（配置）する地域Id。Reference_Dataフォルダー内の「NurtaleNesche_Modding_Region_UUIDs.md」を参照。
⑤"xOffsetFromRegionCenter": 地域の中心から左右にどれだけズラして配置するか。プラスで右、マイナスで左。
⑥"useSpawnRegionCheckpointPosition": trueの場合、地域中心ではなくそのRegionのCheckpoint位置に配置する。この場合"xOffsetFromRegionCenter"は基本効かない。
⑦"yPosOffset": 箱のY座標を上下にズラす値。箱が床に埋まる/浮く時だけ調整。
⑧"dropYVelocity": 箱を開けた時に中身が上方向へ飛び出す強さ。基本変えない。
⑨"dontRespawn": trueの場合、この箱はリスポーン対象にならない。基本falseでよい。
⑩"hasRandomSpawn": trueの場合、ドロップテーブルからランダム抽選する。falseの場合、固定指定を使いやすい。
⑪"dontCurseItem": trueの場合、この箱から出るアイテムが呪われなくなる。
⑫"dontBrokeItem": trueの場合、この箱から壊れたアイテムが出なくなる。
⑬"curseAll": trueの場合、この箱から出るアイテムを全部呪い状態にする。
⑭"dontDropStageCommonDrops": trueの場合、ステージ共通ドロップを混ぜない。"uniqueDropTableIds"だけを使いたい時にtrue。
⑮"isLockedAtStart": trueの場合、箱が最初からロック状態になる。未実装機能なのでfalseのままにする。
⑯"isNeedHammer": trueの場合、箱を開けるのにハンマーが必要になる。未実装機能なのでfalseのままにする。
⑰"openSFXId": 箱を開けた時の効果音Id。基本"sfx.item.box_open"のままでよい。
⑱"uniqueDropTableIds": この箱専用で使うドロップテーブルIdのリスト。特定の中身にしたい時はここを編集する。
⑲"fixedRandomSpawnItemIds": 箱から直接出す固定アイテムIdのリスト。ドロップテーブルを使わず直接指定したい時に使う。"hasRandomSpawn": falseにする必要がある。
⑳"fixedRandomSpawnQuantityList": "fixedRandomSpawnItemIds"の個数リスト。順番を合わせる必要がある。"hasRandomSpawn": falseにする必要がある。
㉑"spawnEquipmentIds": 箱から直接出す装備Idのリスト。装備を固定で入れたい時に使う。"hasRandomSpawn": falseにする必要がある。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。
