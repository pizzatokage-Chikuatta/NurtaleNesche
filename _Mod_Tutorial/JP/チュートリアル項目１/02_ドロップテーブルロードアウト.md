ドロップテーブルロードアウト
================================
ドロップテーブルロードアウトのModを作ります。ドロップテーブルロードアウトは複数のドロップテーブルをまとめて使うためのデータです。
ゲーム内では主に「ステージ共通ドロップ」に使用されます。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_｛個人のニックネーム｝\Lootings\Mod Drop Table Loadouts\Drop Table Loadouts\Stage Common

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Drop Table Loadouts'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて（Visual Studio Code推奨）中身を以下のように変更する。
```json
{
  "experimentalApiVersion": "v1",
  "type": "dropTableLoadouts",
  "entry": "Drop Table Loadouts",
  "overwriteExisting": true
}
```
①今回はドロップテーブルロードアウトのModなので"type"は"dropTableLoadouts"である。
②"entry"は横のフォルダー名と一致しないといけない。

ステップ4
--------
以下のフォルダーから'drop_table_loadout.stage_common.stage_01.json'をコピーし、
StreamingAssets\Lootings\Drop Table Loadouts\Stage Common
以下へ張り付ける。
Mods\Mod_Chikuatta\Lootings\Mod Drop Table Loadouts\Drop Table Loadouts\Stage Common

ステップ5
--------
'drop_table_loadout.stage_common.stage_01.json'の中身を修正する。
①"id": 変えてはいけない。StreamingAssets内でこれと一致するロードアウトが書き換えられる。
②"dropTableIds": 使用するドロップテーブルIdのリスト。
③"drop_table.item_box.stage_01.0"など、アイテム箱専用ドロップテーブルを入れることも可能。
④存在しないドロップテーブルIdを入れると、そのドロップテーブルは機能しない。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

備考
--------
①ステージ共通ドロップを変えたい時は、ドロップテーブル本体だけでなく、このロードアウトがどのドロップテーブルIdを参照しているかも確認する。
②ドロップテーブルを新規追加した場合、そのIdをロードアウトの"dropTableIds"へ追加しないと使われない。

置き換えリストの注意
--------

このファイルは「少し追加するパッチ」ではなく、リスト全体の置き換えです。同じ"id"を上書きする場合、最終的に使われるリストはMod側に書いたリストになります。残したいbuilt-in項目は消さずに残してください。
