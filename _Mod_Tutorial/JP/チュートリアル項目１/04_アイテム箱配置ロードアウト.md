アイテム箱配置ロードアウト
================================
アイテム箱配置ロードアウトのModを作ります。これはステージ開始時に、どのアイテム箱配置Idを使うかを決めるリストです。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_｛個人のニックネーム｝\Lootings\Item Boxes\Mod Stage Initial Spawn Loadouts\Stage Initial Spawn Loadouts

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Stage Initial Spawn Loadouts'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。
```json
{
  "experimentalApiVersion": "v1",
  "type": "itemBoxSpawnLoadouts",
  "entry": "Stage Initial Spawn Loadouts",
  "overwriteExisting": true
}
```
①今回はアイテム箱配置ロードアウトのModなので"type"は"itemBoxSpawnLoadouts"である。
②"entry"は横のフォルダー名と一致しないといけない。

ステップ4
--------
以下のフォルダーから'item_box_spawn_loadout.stage_01.json'をコピーし、
StreamingAssets\Lootings\Item Boxes\Stage Initial Spawn Loadouts
以下へ張り付ける。
Mods\Mod_Chikuatta\Lootings\Item Boxes\Mod Stage Initial Spawn Loadouts\Stage Initial Spawn Loadouts

ステップ5
--------
'item_box_spawn_loadout.stage_01.json'の中身を修正する。
①"itemBoxSpawnLoadoutId": 変えてはいけない。ステージ1のロードアウトを書き換えるためのIdである。
②"itemBoxSpawnIds": ステージ開始時に配置するアイテム箱配置Idのリスト。
③新しく作ったアイテム箱配置をステージに出したい場合、その配置Jsonの"id"をここへ追加する。
④ここにIdを書かないと、アイテム箱配置Jsonが存在していてもステージには出ない。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

備考
--------
①既存の箱を消したい場合は、このロードアウトから対象Idを外す方法と、アイテム箱配置Json側で"enabled": falseにする方法がある。
②複数Modが同じロードアウトを書き換える場合、最後に読み込まれたModが勝つ。

置き換えリストの注意
--------

このファイルは「少し追加するパッチ」ではなく、リスト全体の置き換えです。同じ"id"を上書きする場合、最終的に使われるリストはMod側に書いたリストになります。残したいbuilt-in項目は消さずに残してください。
