アイテム組み合わせ
================================
複数アイテムを組み合わせて別アイテムを作るレシピModを作ります。

この対象は「Use_With_Caution」です。便利ですが壊しやすい項目もあるため、まずは既存データの上書きから始めるのを推奨します。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_Chikuatta\Lootings\Mod Item Combinations\Item Combinations

①"Mod_Chikuatta"は例です。個人のニックネームを使用してください。

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Item Combinations'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。

```json
{
  "experimentalApiVersion": "v1",
  "type": "itemCombinations",
  "entry": "Item Combinations",
  "overwriteExisting": true
}
```

①今回はアイテム組み合わせのModなので"type"は"itemCombinations"である。
②"entry"は横のフォルダー名と一致しないといけない。今回なら"Item Combinations"である。

ステップ4
--------
以下のフォルダーから'item_combination.stage_01_rope_and_hook.json'をコピーし、
StreamingAssets\Lootings\Combinations\Item Combinations
以下へ張り付ける。
Mods\Mod_Chikuatta\Lootings\Mod Item Combinations\Item Combinations

①'item_combination.stage_01_rope_and_hook.json'は例として選んでおり、同じ種類の他ファイルを選んでも手順は同じです。

ステップ5
--------
'item_combination.stage_01_rope_and_hook.json'の中身を修正する。
①"combinationId": レシピid。既存レシピを上書きする場合は変えない。新規なら固有idにする。
②"inputs": 必要な素材アイテムidリスト。順番は基本気にしなくてよいが、既存例に合わせるのが安全。
③"output": 完成するアイテムid。Item Definitionsに存在するidを使う。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

備考
--------
①新規レシピを追加しただけでは使われない場合がある。Item Combination LoadoutにcombinationIdを追加する必要がある。
②存在しないitem idを入れるとレシピが成立しない。
