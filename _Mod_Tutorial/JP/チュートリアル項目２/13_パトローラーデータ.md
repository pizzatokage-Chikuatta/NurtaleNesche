パトローラーデータ
================================
パトローラーのHP、移動速度、検知、タスクなどを変更するModを作ります。

この対象は「Use_With_Caution」です。便利ですが壊しやすい項目もあるため、まずは既存データの上書きから始めるのを推奨します。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_Chikuatta\Patrollers\Mod Patroller Data\Patroller Data\Stage 1

①"Mod_Chikuatta"は例です。個人のニックネームを使用してください。

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Patroller Data'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。

```json
{
  "experimentalApiVersion": "v1",
  "type": "patrollerData",
  "entry": "Patroller Data",
  "overwriteExisting": true
}
```

①今回はパトローラーデータのModなので"type"は"patrollerData"である。
②"entry"は横のフォルダー名と一致しないといけない。今回なら"Patroller Data"である。

ステップ4
--------
以下のフォルダーから'patroller.orc.stage_01.json'をコピーし、
StreamingAssets\Patrollers\Data\Stage 1
以下へ張り付ける。
Mods\Mod_Chikuatta\Patrollers\Mod Patroller Data\Patroller Data\Stage 1

①'patroller.orc.stage_01.json'は例として選んでおり、同じ種類の他ファイルを選んでも手順は同じです。

ステップ5
--------
'patroller.orc.stage_01.json'の中身を修正する。
①"id": 変えてはいけない。既存パトローラーを上書きする場合、このidが一致するデータが書き換えられる。
②"NameInAlphabet": 内部表示用の英名。基本変えない。
③"builtInPrefabAddress": 使用するUnity prefabのアドレス。既存パトローラー上書きでは基本変えない。
④"statusProfileId": 使用できるステータス効果ロードアウトのid。基本変えない。
⑤"actionRuleProfileId": 行動ルールのid。上級者向けなので基本変えない。
⑥"shadowProfileId": 影設定のid。影の見た目を変えたい時だけ変更する。
⑦"animationSetId": 使用するアニメーションセットid。アニメーションModと関係するため最初は変えない。
⑧"showUpStage": 出現ステージ目安。参照・整理用の値として扱う。
⑨"stats": HP、スタミナ、移動速度、インタラクト間隔など。最初に触るならここが安全。
⑩"senses": 検知系。値を壊すと敵がプレイヤーを見つけない/過剰反応するので注意。
⑪"tasks": 行動スケジューラー。タスクid、優先度、cooldownを壊すと敵が止まる可能性がある。
⑫"speechProfileId": セリフ/Speechの参照id。基本変えない。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

備考
--------
①このModは強力だが壊しやすい。最初は"stats"の数値変更だけにするのを推奨。
②"senses"と"tasks"は上級者向け。タスクidや優先度名を間違えるとパトローラーがFallback Idleになる可能性がある。
③新規パトローラー追加はprefab、animationSet、statusProfile、actionRuleなどの整合が必要なので、まずは既存パトローラーの上書きから始める。
