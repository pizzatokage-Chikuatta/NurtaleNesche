装備定義
================================
短剣・クロスボウ・ハンマーなどの装備定義を変更するModを作ります。

この対象は「Use_With_Caution」です。便利ですが壊しやすい項目もあるため、まずは既存データの上書きから始めるのを推奨します。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_Chikuatta\Lootings\Mod Equipment Definitions\Equipment Definitions

①"Mod_Chikuatta"は例です。個人のニックネームを使用してください。

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Equipment Definitions'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。

```json
{
  "experimentalApiVersion": "v1",
  "type": "equipmentDefinitions",
  "entry": "Equipment Definitions",
  "overwriteExisting": true
}
```

①今回は装備定義のModなので"type"は"equipmentDefinitions"である。
②"entry"は横のフォルダー名と一致しないといけない。今回なら"Equipment Definitions"である。

ステップ4
--------
以下のフォルダーから'equipment.dagger.json'をコピーし、
StreamingAssets\Lootings\Items And Equipments\Equipment Definitions
以下へ張り付ける。
Mods\Mod_Chikuatta\Lootings\Mod Equipment Definitions\Equipment Definitions

①'equipment.dagger.json'は例として選んでおり、同じ種類の他ファイルを選んでも手順は同じです。

ステップ5
--------
'equipment.dagger.json'の中身を修正する。
①"id": 既存装備を上書きする場合は変えてはいけない。
②"NameInAlphabet": 内部表示用英名。
③"showUpStage": 出現ステージ目安。
④"builtInPrefabAddress": 装備prefabの参照。基本変えない。
⑤"assetBundleFileName"/"prefabAssetPathInsideBundle"/"assemblyFileName"/"logicTypeName": 外部コード/外部prefab用。通常のJSON Modでは触らない。
⑥"spriteIds": 装備の表示sprite id。displayPristine/displayWorn/displayBrokenなどのkeyがある。
⑦"extras": 装備ごとの追加設定。空でもよい。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

備考
--------
①新規装備はUI、sprite、入手経路、実際の挙動が必要になるため上級者向け。最初は既存装備の見た目や表示設定の上書き推奨。
②装備の攻撃挙動そのものはJSONだけでは作れない場合がある。

ゲーム内に出すための依存関係
--------

新しいitem/equipment定義を読み込ませるだけでは、ゲーム内に自然出現しません。Drop Table、Item Box、shop/loadout、または他のゲームデータから参照される必要があります。名前、sprite、UIも別データで合わせる必要がある場合があります。
