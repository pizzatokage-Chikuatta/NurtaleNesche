装備UI
================================
装備がUI上でどのキー枠・アクション名として表示されるかを変更するModを作ります。

この対象は「Use_With_Caution」です。便利ですが壊しやすい項目もあるため、まずは既存データの上書きから始めるのを推奨します。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_Chikuatta\Lootings\Mod Equipment UI\Equipment UI

①"Mod_Chikuatta"は例です。個人のニックネームを使用してください。

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Equipment UI'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。

```json
{
  "experimentalApiVersion": "v1",
  "type": "equipmentUi",
  "entry": "Equipment UI",
  "overwriteExisting": true
}
```

①今回は装備UIのModなので"type"は"equipmentUi"である。
②"entry"は横のフォルダー名と一致しないといけない。今回なら"Equipment UI"である。

ステップ4
--------
以下のフォルダーから'dagger.json'をコピーし、
StreamingAssets\Lootings\Items And Equipments\Equipment UI
以下へ張り付ける。
Mods\Mod_Chikuatta\Lootings\Mod Equipment UI\Equipment UI

①'dagger.json'は例として選んでおり、同じ種類の他ファイルを選んでも手順は同じです。

ステップ5
--------
'dagger.json'の中身を修正する。
①"equipmentId": 対象装備id。装備定義の"id"と一致させる。
②"isTrinket": Trinket扱いするかどうか。通常装備はfalse。
③"hudKeyIndex": HUD上の表示枠番号。被ると表示が競合する可能性がある。
④"quickUseAction": 短押し時の表示/操作名。
⑤"longPressAction": 長押し時の表示/操作名。
⑥"escapeToolId": 拘束脱出に使う道具として扱う場合のid。基本既存のまま。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

備考
--------
①Equipment UIは装備の挙動そのものを作るものではない。表示と入力案内側の設定である。
②equipmentIdが存在しないと意味がないので、装備定義とセットで考える。
