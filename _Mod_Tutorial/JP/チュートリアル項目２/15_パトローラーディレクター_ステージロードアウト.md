パトローラーディレクター・ステージロードアウト
================================
どのスポーンポイントがどのパトローラーディレクターを使うかを変更するModを作ります。

この対象は「Use_With_Caution」です。便利ですが壊しやすい項目もあるため、まずは既存データの上書きから始めるのを推奨します。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_Chikuatta\Patrollers\Mod Patroller Director Stage Loadouts\Patroller Director Stage Loadouts

①"Mod_Chikuatta"は例です。個人のニックネームを使用してください。

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Patroller Director Stage Loadouts'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。

```json
{
  "experimentalApiVersion": "v1",
  "type": "patrollerDirectorStageLoadouts",
  "entry": "Patroller Director Stage Loadouts",
  "overwriteExisting": true
}
```

①今回はパトローラーディレクターのステージロードアウトのModなので"type"は"patrollerDirectorStageLoadouts"である。
②"entry"は横のフォルダー名と一致しないといけない。今回なら"Patroller Director Stage Loadouts"である。

ステップ4
--------
以下のフォルダーから'patroller_director_loadout.stage_03.json'をコピーし、
StreamingAssets\Patrollers\Spawn Directors\Stage Loadouts
以下へ張り付ける。
Mods\Mod_Chikuatta\Patrollers\Mod Patroller Director Stage Loadouts\Patroller Director Stage Loadouts

①'patroller_director_loadout.stage_03.json'は例として選んでおり、同じ種類の他ファイルを選んでも手順は同じです。

ステップ5
--------
'patroller_director_loadout.stage_03.json'の中身を修正する。
①"id": 変えてはいけない。ステージごとのロードアウトidである。
②"assignments": スポーンポイントとディレクターidの対応リスト。
③"spawnPointId": ステージ内のディレクタースポーンポイントid。存在しないidは動かない。
④"directorId": 使用するパトローラーディレクターid。Patroller Directors側の"id"と一致させる。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

備考
--------
①このMod単体では敵の中身は変わらない。中身を変える場合はパトローラーディレクター側も編集する。
②spawnPointIdはステージ内に存在するものだけ使える。名前を適当に作っても新しいスポーンポイントは生えない。

置き換えリストの注意
--------

このファイルは「少し追加するパッチ」ではなく、リスト全体の置き換えです。同じ"id"を上書きする場合、最終的に使われるリストはMod側に書いたリストになります。残したいbuilt-in項目は消さずに残してください。
