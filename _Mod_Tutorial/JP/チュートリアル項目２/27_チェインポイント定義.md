チェインポイント定義
================================
チェインポイント本体の定義を変更するModを作ります。

この対象は「Use_With_Caution」です。便利ですが壊しやすい項目もあるため、まずは既存データの上書きから始めるのを推奨します。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_Chikuatta\Chainpoints\Mod Chainpoint Definitions\Chainpoint Definitions

①"Mod_Chikuatta"は例です。個人のニックネームを使用してください。

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Chainpoint Definitions'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。

```json
{
  "experimentalApiVersion": "v1",
  "type": "chainPointDefinitions",
  "entry": "Chainpoint Definitions",
  "overwriteExisting": true
}
```

①今回はチェインポイント定義のModなので"type"は"chainPointDefinitions"である。
②"entry"は横のフォルダー名と一致しないといけない。今回なら"Chainpoint Definitions"である。

ステップ4
--------
以下のフォルダーから'chainpoint.kabe_manguri.json'をコピーし、
StreamingAssets\ChainPoints\Definitions
以下へ張り付ける。
Mods\Mod_Chikuatta\Chainpoints\Mod Chainpoint Definitions\Chainpoint Definitions

①'chainpoint.kabe_manguri.json'は例として選んでおり、同じ種類の他ファイルを選んでも手順は同じです。

ステップ5
--------
'chainpoint.kabe_manguri.json'の中身を修正する。
①"id": chainpoint id。既存チェインポイントを上書きする場合は変えない。
②"statusId": 捕獲時に付与するStatus Effect ID。
③"typeEnum": チェインポイントの種類。基本変えない。
④"accessScope": 使用可能範囲。基本既存のまま。
⑤"isNeedLockPickToEscape": 脱出にロックピックが必要かどうか。
⑥"runtimeSpawnable": runtime spawn可能かどうか。falseのものは配置Modで出す対象にしない。
⑦"stateAnimatorBindings": チェインポイント状態とAnimator stateの対応。上級者向け。
⑧"interactionLoadout": インタラクション設定のid。基本変えない。
⑨"addressableKey": Unity側のprefab/addressable参照。基本変えない。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

備考
--------
①チェインポイントはゲーム進行や捕獲状態と直結するため、定義変更は慎重に行う。
②配置だけしたい場合は、先にChainpoint Spawn / Chainpoint Spawn Loadoutチュートリアルを使うのが安全。
③runtimeSpawnableがfalseのチェインポイントは、Modでランタイム配置する対象にしない。
