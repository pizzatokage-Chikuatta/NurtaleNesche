スプライト
================================
アイテムや装備などの通常スプライト参照を追加/変更するModを作ります。

この対象は「Use_With_Caution」です。便利ですが壊しやすい項目もあるため、まずは既存データの上書きから始めるのを推奨します。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_Chikuatta\Sprites\Mod Sprites\Sprites\Equipment

①"Mod_Chikuatta"は例です。個人のニックネームを使用してください。

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Sprites'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。

```json
{
  "experimentalApiVersion": "v1",
  "type": "sprites",
  "entry": "Sprites",
  "overwriteExisting": true
}
```

①今回はスプライトのModなので"type"は"sprites"である。
②"entry"は横のフォルダー名と一致しないといけない。今回なら"Sprites"である。

ステップ4
--------
以下のフォルダーから'sprite.equipment.dagger_healthy.json'をコピーし、
StreamingAssets\Sprites\Equipment
以下へ張り付ける。
Mods\Mod_Chikuatta\Sprites\Mod Sprites\Sprites\Equipment

①'sprite.equipment.dagger_healthy.json'は例として選んでおり、同じ種類の他ファイルを選んでも手順は同じです。

ステップ5
--------
'sprite.equipment.dagger_healthy.json'の中身を修正する。
①"id": sprite id。既存spriteを上書きする場合は変えない。
②"builtInSpriteAddress": 画像ファイル参照。Mod画像を使う場合はMod側のpng参照に合わせる。
③"pixelsPerUnit": 画像の大きさ基準。値を変えるとゲーム内表示サイズが変わる。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

備考
--------
①これは通常のアイテム/装備/UIスプライト用で、アニメーション用PNG差し替えとは別物。
②画像を差し替える時は、sprite idを参照しているItem DefinitionsやEquipment Definitionsとの対応を確認する。
③pixelsPerUnitを大きく変えると表示サイズが崩れやすい。まずは既存値のまま推奨。

ファイル参照の流れ
--------

ファイルを登録することと、ゲーム内で使わせることは別です。例えば音声は、audio file -> clip ID -> classification/track/loadout -> gameplay reference のように参照がつながります。
