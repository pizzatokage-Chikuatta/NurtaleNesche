アイテム定義
================================
アイテム本体の定義を変更するModを作ります。

この対象は「Use_With_Caution」です。便利ですが壊しやすい項目もあるため、まずは既存データの上書きから始めるのを推奨します。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_Chikuatta\Lootings\Mod Item Definitions\Item Definitions\MISC

①"Mod_Chikuatta"は例です。個人のニックネームを使用してください。

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Item Definitions'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。

```json
{
  "experimentalApiVersion": "v1",
  "type": "itemDefinitions",
  "entry": "Item Definitions",
  "overwriteExisting": true
}
```

①今回はアイテム定義のModなので"type"は"itemDefinitions"である。
②"entry"は横のフォルダー名と一致しないといけない。今回なら"Item Definitions"である。

ステップ4
--------
以下のフォルダーから'item.biscuit.json'をコピーし、
StreamingAssets\Lootings\Items And Equipments\Item Definitions\MISC
以下へ張り付ける。
Mods\Mod_Chikuatta\Lootings\Mod Item Definitions\Item Definitions\MISC

①'item.biscuit.json'は例として選んでおり、同じ種類の他ファイルを選んでも手順は同じです。

ステップ5
--------
'item.biscuit.json'の中身を修正する。
①"id": 既存アイテムを上書きする場合は変えてはいけない。新規アイテムなら固有idにする。
②"NameInAlphabet": 内部表示用英名。
③"showUpStage": 出現ステージ目安。
④"brokenChance": 壊れた状態になる確率。0なら壊れない。
⑤"itemSelectionRequestId": アイテム使用時に選択対象が必要な場合の参照id。上級者向け。
⑥"builtInPrefabAddress": Unity prefab参照。既存上書きでは基本変えない。
⑦"assetBundleFileName"/"prefabAssetPathInsideBundle"/"assemblyFileName"/"logicTypeName": コード/AssetBundle連携用。JSONだけの簡単Modでは基本nullのまま。
⑧"spriteIds": 表示に使うsprite id。Sprites Modや既存sprite idと合わせる。
⑨"extras": アイテムごとの追加設定。例: dropSFXIdなど。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

備考
--------
①新しいアイテムを作る場合、見た目、名前、ドロップテーブル、場合によっては挙動も別途必要になる。まずは既存アイテムの上書き推奨。
②アイテムの実際の挙動はコード側にあるものも多い。JSONだけで完全に新しい効果を発明できるわけではない。

ゲーム内に出すための依存関係
--------

新しいitem/equipment定義を読み込ませるだけでは、ゲーム内に自然出現しません。Drop Table、Item Box、shop/loadout、または他のゲームデータから参照される必要があります。名前、sprite、UIも別データで合わせる必要がある場合があります。
