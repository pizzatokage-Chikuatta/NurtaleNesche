パトローラーディレクター
================================
ステージ内で追加パトローラーを時間経過で出すためのディレクターModを作ります。

この対象は「Use_With_Caution」です。便利ですが壊しやすい項目もあるため、まずは既存データの上書きから始めるのを推奨します。

ステップ1
--------
'Mods'の中に以下のフォルダー階層を作る。
Mods\Mod_Chikuatta\Patrollers\Mod Patroller Directors\Patroller Directors\Stage 3

①"Mod_Chikuatta"は例です。個人のニックネームを使用してください。

ステップ2
--------
Mods\mod.json templatesから'mod.template_folder_backed.json'をコピーし、'Mod Patroller Directors'の中に張り付け、名前を'mod.json'へ変更する。

ステップ3
--------
mod.jsonを開いて中身を以下のように変更する。

```json
{
  "experimentalApiVersion": "v1",
  "type": "patrollerDirectors",
  "entry": "Patroller Directors",
  "overwriteExisting": true
}
```

①今回はパトローラーディレクターのModなので"type"は"patrollerDirectors"である。
②"entry"は横のフォルダー名と一致しないといけない。今回なら"Patroller Directors"である。

ステップ4
--------
以下のフォルダーから'patroller_director.01.stage_03.json'をコピーし、
StreamingAssets\Patrollers\Spawn Directors\Stage 3
以下へ張り付ける。
Mods\Mod_Chikuatta\Patrollers\Mod Patroller Directors\Patroller Directors\Stage 3

①'patroller_director.01.stage_03.json'は例として選んでおり、同じ種類の他ファイルを選んでも手順は同じです。

ステップ5
--------
'patroller_director.01.stage_03.json'の中身を修正する。
①"id": 変えてはいけない。ロードアウトからこのidで参照される。
②"patrolRegionIds": スポーンしたパトローラーが巡回候補にするRegion UUIDリスト。Region UUIDはReference_Dataフォルダー内の「NurtaleNesche_Modding_Region_UUIDs.md」を参照。
③"startActive": trueならステージ開始時から有効。falseならイベント等で有効化されるまで動かない。
④"autoActivation": trueなら一定時間後に自動で有効化される。
⑤"autoActivationDelay": autoActivation時の有効化までの秒数。
⑥"maxAliveFromThisSpawner": このディレクターから同時に生きていてよい最大数。
⑦"maxAliveOnFloor": フロア全体の敵数制限。多すぎるとスポーンしない。
⑧"spawnCooldown": 次のスポーンまでの基本秒数。
⑨"spawnJitterX": スポーン位置の左右ブレ幅。
⑩"spawnCountChoices": 1回のスポーンで何体出すかの候補。
⑪"patrollers": 出すパトローラーidとweight。weightが大きいほど選ばれやすい。

最後にゲームのメインメニュー -> Mods -> Mod Reportで正しく認識されているか確認することで完了。

備考
--------
①Stage 3/5/6など、ディレクターが使われているステージ向け。ロードアウトに入っていないディレクターは使われない。
②敵数を増やしすぎると難易度と処理負荷が大きく変わるので注意。
③patrollerIdはPatroller IDs参照データから選ぶ。存在しないidを入れるとスポーンできない。
