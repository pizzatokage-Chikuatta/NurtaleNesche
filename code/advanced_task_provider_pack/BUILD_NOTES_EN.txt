Developer-only note:

Normal JSON mod authors can ignore this folder. It is useful only in a build that intentionally supports loading mod assemblies from `Mods`.

If that pipeline is enabled, the code side must provide a compiled assembly that contains a concrete task provider type. The optional `*.task_providers.json` manifest then maps a public provider id to that loaded C# type.

Recommended v1 experimental authoring path:

- inherit from `NurtaleNesche.Modding.Patrollers.Tasks.TaskSetCore` or `TaskProviderCore`
- use `NurtaleNesche.Modding.Patrollers.Tasks.TaskProviderIdAttribute`
- use `NurtaleNesche.Modding.Patrollers.Tasks.TaskSequenceBuilder` or `TaskInfo` to construct the current runtime task container
- use typed helpers from `NurtaleNesche.Modding.Registries.ModRegistryExtensions`, such as `RegisterTaskProvider<T>`, inside registrar classes if manual registration is needed
- remember that State/StateArgs usage is still experimental runtime coupling

Without the matching assembly, the manifest is only data and the loader will warn that the type was not found.
