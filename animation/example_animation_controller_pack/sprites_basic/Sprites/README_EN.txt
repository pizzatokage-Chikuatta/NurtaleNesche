Type: sprites
Example file: sprite.example.training_dagger.json

What this data does:
It registers a normal sprite id that UI, equipment, or item data can reference.

Fields shown in the example:

- `id`: canonical sprite id.
- `builtInSpriteAddress`: built-in sprite asset path. The example points to the healthy dagger sprite.
- `pixelsPerUnit`: sprite pixel scale used when loading/rendering this sprite.
