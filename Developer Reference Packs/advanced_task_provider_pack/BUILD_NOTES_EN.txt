Advanced Task Provider Reference Pack - Build Notes

This file is a short reference note only.
The complete tutorial lives in:

../../_Mod_Tutorial/EN/Tutorial_Group_5

Minimum concept:

1. Create a normal C# class library project.
2. Add references to the same-version game DLLs:
   - NurtaleNesche.Modding.Abstractions.dll
   - NurtaleNesche.Runtime.dll
   - UnityEngine.CoreModule.dll
   - Newtonsoft.Json.dll
   - netstandard.dll if your compiler requires it
3. Copy ExampleTaskProvider.cs.txt into your project as ExampleTaskProvider.cs.
4. Optionally copy ExampleTaskProviderRegistrar.cs.txt as ExampleTaskProviderRegistrar.cs.
5. Build your DLL.
6. Put the DLL and example.task_providers.json next to each other in the final mod folder.
7. Add the provider ID to Patroller Data tasks.providers.

Important:

- The game loads compiled DLLs. It does not compile .cs files from Mods.
- The manifest maps an ID to a C# type. It does not create behavior by itself.
- The Patroller Data provider entry activates the provider for that patroller.
- ApplyConfig reads the config object from the Patroller Data provider entry.
- Use Group 5 tutorials for full step-by-step setup and troubleshooting.

Experimental warning:

Task providers, task states, state args, and task containers are v1 experimental surfaces. Rebuild and retest code mods when the game updates.
